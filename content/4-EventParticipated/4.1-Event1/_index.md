# Summary Report: "FCAJ Community Day"

### Event Information

- **Time:** 08:30 - 12:00, 19/05/2026
- **Location:** 26th Floor, Bitexco Tower, District 1
- **Role:** Intern

### Speakers

- **Thinh Nguyen** – Speaker, "Context Is Everything"
- **Long Huynh** – Speaker, LLM & Multi-Agent Systems
- **Thu Dang** – Speaker, Developer Experience & Hackathon Insights

### Key Highlights

#### Context Is Everything

- Why AI models "hallucinate" or go off-track when context is insufficient
- Practical techniques for managing multi-context scenarios in real-world projects
- Core takeaway: **a great prompt without solid context still underperforms**

#### Deep-Dive: LLM Temperature Settings

- The truth behind setting **Temperature = 0** in LLMs and common misconceptions
- Trade-offs between determinism and output quality in production use cases

#### Multi-Agent Systems in Practice

- How a Multi-Agent architecture operates in a real startup credit-scoring use case
- Criteria for deciding between a single Agent and a Multi-Agent System depending on system complexity

#### Developer Stories: 36-Hour Deadline at LotusHacks

- First-hand account from developers on surviving a 36-hour hackathon deadline
- Lessons on resilience, prioritization, and staying focused under intense time pressure

### Key Takeaways

#### Working with AI Effectively

- **Context beats prompting**: feeding the right context to AI produces far better results than generic prompt engineering alone
- Practical techniques for injecting context to improve AI output quality in real projects

#### Technical Depth

- Clear criteria for choosing between a single Agent and a Multi-Agent System in large-scale systems
- Practical LLM optimization tips that can be applied directly to personal projects or apps currently in development

#### Enterprise Reality Check

- Clear distinction between building an AI demo and integrating AI into an enterprise system
- Enterprise AI adoption requires attention to **security, compliance**, and **cost optimization** (e.g., using CloudFront to reduce delivery costs)

### Applying to Work

- **Improve prompting practice**: prioritize context-building over prompt wording alone in current and future projects
- **Evaluate Agent architecture**: assess whether a single Agent or Multi-Agent System fits the complexity of a given task
- **Apply LLM optimization tips**: bring practical Temperature and configuration insights into ongoing development work
- **Consider enterprise constraints early**: factor in security, compliance, and cost (e.g., CDN caching) when moving AI features from demo to production

### Event Experience

Attending **"FCAJ Community Day"** was a valuable opportunity to hear real-world case studies on Cloud and AI directly from practitioners close in age and experience. Key experiences included:

#### Learning from peer practitioners
- The session on **"Context Is Everything"** stood out the most, offering a clear, relatable explanation of why AI often underperforms without sufficient context — a lesson directly applicable to handling multi-context scenarios in real projects.
- The technical deep-dives, including the truth behind **Temperature = 0** and how **Multi-Agent Systems** operate in a startup credit-scoring pipeline, gave a much deeper, more grounded understanding than surface-level tutorials.

#### Hands-on and practical insights
- Learned concrete criteria for deciding between a single Agent and a Multi-Agent System based on system complexity.
- Picked up practical LLM optimization tips that can be applied immediately to personal projects or apps currently in development.

#### Real-world perspective
- Hearing developers share their experience surviving a **36-hour deadline at LotusHacks** was genuinely inspiring and reinforced the determination needed to push through high-pressure situations.
- Gained a clearer view of the gap between a polished AI demo and a production-ready enterprise AI system — one that must account for **security, compliance, and cost optimization** such as using **CloudFront** to manage delivery costs.

#### Lessons learned
- **Context is the real differentiator** in AI performance, more so than prompt phrasing alone.
- Choosing between single-Agent and Multi-Agent architectures should be driven by actual system complexity, not by default habit.
- Moving AI from demo to enterprise production requires deliberate planning around security, compliance, and cost — it is a fundamentally different challenge than building a prototype.
